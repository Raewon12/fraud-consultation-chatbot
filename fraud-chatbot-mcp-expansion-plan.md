# 중고거래 사기 피해 상담 챗봇 — MCP 확장 계획

## 현재 시스템 구조

```
사용자 질문
  ↓
의도 분류 (counseling / form_writing)
  ↓
RAG 검색 (Chroma + Cross-Encoder Reranking)
  ↓
텍스트 답변 (gpt-4o-mini)
```

현재는 사용자의 질문에 대해 **텍스트 기반 상담**만 제공하는 구조이다.
MCP(Model Context Protocol) 서버를 연동하면, 텍스트 답변을 넘어 **실제 행동을 수행하는 AI 에이전트**로 확장할 수 있다.

---

## MCP 확장 후 목표 구조

```
사용자 질문
  ↓
의도 분류 (counseling / form_writing / action)
  ↓
RAG 검색 → 답변 생성 + MCP 도구 호출
                          │
          ┌───────────────┼───────────────┐
          │               │               │
    PDF 생성 MCP    위치 검색 MCP    증거 관리 MCP
    (고소장 작성)   (경찰서 찾기)    (체크리스트)
```

## MCP 확장 항목 상세

### 1순위: 법률 서식 PDF 자동 생성

| 항목 | 내용 |
|------|------|
| **현재** | "고소장에는 이런 내용이 들어가야 합니다" (텍스트 설명) |
| **MCP 적용 후** | 사용자 정보를 수집하여 실제 고소장 PDF 파일을 생성·다운로드 |
| **대상 서식** | 고소장, 내용증명, 고발장, 민사소장 |
| **활용 데이터** | `data/legal_info/templates/` (이미 존재) |
| **구현 난이도** | ★★☆☆☆ (템플릿이 이미 있어서 연결만 하면 됨) |

**예상 사용자 경험:**

```
사용자: "고소장 써줘"
에이전트: "고소장 작성에 필요한 정보를 여쭤볼게요.
          1. 피고소인 이름(또는 닉네임)이 무엇인가요?
          2. 거래 날짜는 언제인가요?
          3. 피해 금액은 얼마인가요?
          4. 입금한 계좌 정보를 알려주세요."
사용자: (정보 입력)
에이전트: "고소장이 완성되었습니다. PDF를 다운로드하세요."
          → [고소장_20260406.pdf 다운로드]
```

**MCP 서버 구현 예시 (Python):**

```python
from mcp import Server
from reportlab.lib.pagesizes import A4
from reportlab.pdfgen import canvas

mcp = Server("fraud-pdf-generator")

@mcp.tool()
def generate_complaint_pdf(
    victim_name: str,
    suspect_name: str,
    transaction_date: str,
    amount: int,
    account_info: str,
    description: str
) -> dict:
    """피해자 정보를 받아 고소장 PDF를 생성합니다."""
    
    filename = f"고소장_{transaction_date}.pdf"
    # PDF 생성 로직
    c = canvas.Canvas(filename, pagesize=A4)
    # ... 템플릿에 데이터 채우기 ...
    c.save()
    
    return {"file_path": filename, "status": "생성 완료"}
```

| 항목 | 내용 |
|------|------|
| **현재** | 계좌 지급정지 방법을 텍스트로 안내 |
| **MCP 적용 후** | 은행명 입력 시 해당 은행 긴급 전화번호 즉시 조회 + 112 신고 스크립트 자동 생성 |
| **활용 데이터** | `data/emergency_guide/account_freeze.json`, `report_contacts.json` |
| **구현 난이도** | ★☆☆☆☆ (기존 데이터 활용, 포맷팅만 추가) |

**예상 사용자 경험:**

```
사용자: "방금 사기당했어요! 국민은행으로 송금했어요"
에이전트: "⚠️ 긴급 조치가 필요합니다!

          [즉시 전화하세요]
          • 국민은행 사고신고: 1588-9999 (24시간)
          • 경찰청: 112
          
          [112 신고 시 이렇게 말하세요]
          '중고거래 사기를 당했습니다.
           국민은행 계좌 XXX-XXXX-XXXX로 
           XX만원을 송금했고, 
           지급정지를 요청합니다.'
          
          [다음 단계]
          → 통화 후 '지급정지 완료했어요'라고 알려주세요.
            증거 수집 체크리스트를 안내해드리겠습니다."
```

**MCP 서버 구현 예시:**

```python
@mcp.tool()
def get_emergency_contacts(bank_name: str) -> dict:
    """은행명을 받아 긴급 신고 전화번호와 신고 스크립트를 반환합니다."""
    
    bank_contacts = {
        "국민은행": "1588-9999",
        "신한은행": "1577-8000",
        "우리은행": "1588-5000",
        # ...
    }
    
    return {
        "bank_phone": bank_contacts.get(bank_name, "해당 은행 정보 없음"),
        "police": "112",
        "cyber_crime": "182",
        "script": f"중고거래 사기 피해로 {bank_name} 계좌 지급정지를 요청합니다."
    }
```

## 구현 로드맵

```
[1단계] PDF 생성 MCP (1순위)
  │  기존 템플릿 활용 → 빠른 구현 가능
  │  "텍스트 챗봇" → "서류 생성 에이전트"로 전환되는 핵심
  ▼
[2단계] 계좌 지급정지 + 증거 관리 MCP (2~3순위)
  │  기존 JSON 데이터 활용 → 중간 난이도
  │  피해 직후 골든타임 대응력 강화
  ▼
[3단계] 위치 검색 + 판례 검색 MCP (4~5순위)
  │  외부 API 연동 필요 → 높은 난이도
  │  서비스 완성도 향상
  ▼
[4단계] 타임라인 생성 MCP (6순위)
     종합 에이전트 완성
```

## 기존 코드 변경 최소화 전략

MCP 서버는 **기존 RAG 파이프라인과 독립적으로 동작**한다.
기존 `rag.py`의 구조를 크게 바꿀 필요 없이, 의도 분류 단계에서 MCP 도구 호출 분기만 추가하면 된다.

```python
# rag.py 수정 최소화 예시
def classify_intent(question: str) -> str:
    # 기존
    if is_form_writing(question):
        return "form_writing"
    
    # MCP 추가
    if is_pdf_generation(question):
        return "pdf_generation"  # → MCP 서버 호출
    if is_emergency(question):
        return "emergency_action"  # → MCP 서버 호출
    
    return "counseling"  # 기존 RAG 상담
```

## 핵심 요약

| 구분 | 현재 (챗봇) | MCP 적용 후 (에이전트) |
|------|-------------|----------------------|
| 고소장 | "이렇게 쓰세요" (설명) | PDF 파일 자동 생성 |
| 긴급 대응 | "112에 신고하세요" (안내) | 은행 전화번호 + 신고 스크립트 제공 |
| 증거 수집 | "이런 증거가 필요해요" (나열) | 체크리스트 추적 + 누락 알림 |
| 경찰서 | "가까운 경찰서에 가세요" (일반) | 위치 기반 경찰서 검색 |
| 판례 | 사례 10건 기반 상담 | 실시간 판례 검색 + 통계 |

> **1순위 PDF 생성 MCP 하나만 추가해도,
> "알려주는 챗봇"에서 "해주는 에이전트"로 확 달라진다.**
