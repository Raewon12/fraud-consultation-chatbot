## FastAPI 백엔드 + SSE 스트리밍 구현

### 백엔드 (`api.py`)
- FastAPI 서버 생성, rag.py의 FraudRAGAssistant를 API로 감쌈
- **REST API**: `POST /api/chat` — 답변 한 번에 반환
- **SSE 스트리밍**: `POST /api/chat/stream` — 글자 단위 실시간 스트리밍
- 세션 관리: 세션별 독립 대화 기록, 무거운 리소스(벡터 저장소, reranker)는 공유
- CORS 설정 (localhost:3000 허용)
- 추가 엔드포인트: `/api/chat/reset` (대화 초기화), `/api/health` (상태 확인)

### 프론트엔드 (`frontend/app/chat/page.tsx`)
- 더미 응답 제거 → FastAPI SSE 엔드포인트에 연결
- 글자 단위 스트리밍으로 실시간 타이핑 효과 구현
- 세션 ID 자동 생성 (crypto.randomUUID)
- 에러 핸들링 추가

### 연동 테스트 결과
- 상담 모드: "당근마켓에서 에어팟 5만원에 샀는데 입금하고 나서 연락이 안 돼요" → 정상 응답
- 서식작성 모드: "고소장 작성해줘" → 필요 정보 요청 → 정보 입력 → 고소장 초안 생성 완료
- 멀티턴 대화, 의도 분류(상담/서식작성) 정상 동작 확인

### 고소장 템플릿 개선
- `플랫폼 닉네임` 필드: required: true로 변경 (피고소인 특정에 중요)
- `증거 자료` 섹션: 사용자가 실제 확보한 증거를 반영하도록 필드 추가
